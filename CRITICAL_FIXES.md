# Критические исправления: экспорт/импорт и расчет RIR

## Исправленные проблемы

### 1. Экспорт базы данных (db.js)

**Проблема:** Использование spread оператора `String.fromCharCode(...new Uint8Array(data))` вызывало ошибки при больших массивах (stack overflow).

**Решение:**
- Заменен spread оператор на цикл для преобразования в base64
- Используется безопасный метод: `for (let i = 0; i < bytes.length; i++) bin += String.fromCharCode(bytes[i])`
- Применено как для несжатых, так и для сжатых данных

**Файл:** `db.js`, функция `exportDatabase()`

### 2. Расчет RIR при редактировании сета (app.js)

**Проблема:** RIR рассчитывался от e1RM вместо TM, что давало заниженные значения (TM ≈ 90% от e1RM).

**Решение:**
- RIR теперь рассчитывается от TM: `estRIR(tm, weight, reps)`
- TM загружается из базы данных перед расчетом
- Добавлено автообновление TM после редактирования сета
- Обновляется TM в карточке упражнения, если она открыта

**Файл:** `app.js`, функция `editSet()`

### 3. Обновление TM в карточке после редактирования

**Добавлено:**
- После редактирования сета обновляется TM в карточке упражнения, если она открыта
- Обновляется тултип TM с информацией о лучшем e1RM за 4 недели
- Обеспечивается синхронизация данных между историей и карточкой

## Чек-лист для тестирования

1. **Открыть приложение** → убедиться, что план загрузился
2. **Ввести несколько сетов** → сохранить черновик/сессию
3. **Экспорт**: кнопка "📥 Экспорт" → скачивается `meso-backup-YYYY-MM-DD-…workouts.json`
4. **(Опционально)** Очистить IndexedDB в DevTools, перезагрузить страницу
5. **Импорт**: кнопка "📤 Импорт" → выбрать скачанный файл → данные восстанавливаются
6. **Проверить дашборд и историю** → данные должны совпадать с "до экспорта"
7. **Редактировать сет из истории** → RIR должен рассчитываться от TM, а не от e1RM
8. **Проверить TM в карточке** → должен обновиться после редактирования сета

## Формат бэкапа

```json
{
  "version": 2,
  "data": "base64_encoded_data",
  "timestamp": "2024-01-01T00:00:00.000Z",
  "compressed": true,
  "originalSize": 12345,
  "compressedSize": 5432
}
```

- `version`: версия формата (2)
- `data`: base64-кодированные данные SQLite (сжатые или несжатые)
- `timestamp`: время создания бэкапа
- `compressed`: флаг сжатия (true/false)
- `originalSize`: размер несжатых данных (байты)
- `compressedSize`: размер сжатых данных (байты)
