
# ТЗ: GS_Trainer — MVP (PWA на GitHub Pages + Google Apps Script + Google Sheets)

**Цель:** собрать бесплатный мобильный MVP (iPhone 11 first), где пользователь выбирает **Неделю (1–4)** и **День (1–4)**, видит **список упражнений дня** с **рекомендованным весом (TM×%)**, вводит **факт** (повторы/вес/RIR/заметка) и **сохраняет** всё в лист **Log** одним нажатием. Вся бизнес‑логика — на сервере (Apps Script).

---

## 1) Объем работ (Scope)

### Входит (MVP)
- **Фронт (PWA, GitHub Pages):** один экран, чистые HTML/CSS/JS (без фреймворков).
- **Бэкенд (Apps Script Web App):** два эндпойнта — **GET /day** и **POST /log**.
- **Структура Google Sheet:** листы **Settings, Plan, Log, TMs, Archive**.
- **Анти‑дубликаты** при сохранении: `(Exercise|Set)` в окне **±60 сек**.
- **Инкрементальное обновление TMs** по новому e1RM.
- **Warm‑up** в ответе `/day` для основных упражнений.
- Локализация **ru**; единицы по умолчанию **kg**.

### Не входит (после MVP)
- Полный офлайн/кеш (service worker — заглушка).
- Пуши, HealthKit/Watch, авторизация/роли.
- PDF/экспорт, сложная аналитика/дашборды.

---

## 2) Результаты (Deliverables)

- Репозиторий с каталогами `/pwa`, `/gas`, (опц.) `/docs` и `README.md`.
- Фронт: статическая PWA‑страница с формой и кнопкой **Save**.
- Бэкенд: деплой Apps Script как **Web app** (Anyone with the link).
- Рабочая связка: ввод → POST `/log` → строки появляются в **Log**, **TMs** обновляется при PR.

## 4) Схема данных (Google Sheets)

**Settings** (B:C, key/value):  
`UNITS=kg`, `STEP_KG=2.5`, `STEP_LB=5`, `BAR_WEIGHT=20`, `ARCHIVE_DAYS=60`

**Plan** (A:J):  
`Key | Week | Day | Exercise | Set # | Rep Min | Rep Max | RIR | Prescribed | PctTM`

**Log** (A:O):  
`Timestamp | Week# | Day# | Exercise | Set # | Rep Min | Rep Max | RIR план | Prescribed | Actual Reps | Actual Weight | Actual RIR | e1RM | TM (90%) | Notes`

**TMs**:  
`Exercise | last e1RM | TM (90%) | Updated`

**Archive**: копия структуры **Log**.

## 5) API (контракты)

### 5.1 GET `/day?week=1&day=3`
**Ответ 200 (application/json):**
```json
{
  "week": 1,
  "day": 3,
  "units": "kg",
  "step": 2.5,
  "rows": [
    {
      "exercise": "Жим лёжа (штанга)",
      "set": 1,
      "repMin": 2,
      "repMax": 4,
      "rirPlan": 1,
      "pct": 0.88,
      "prescribed": null,
      "tm": 112.5,
      "recommended": 100.0
    }
  ],
  "warmup": {
    "Жим лёжа (штанга)": [
      {"label":"Пустой гриф","pct":null,"reps":"10/5","weight":20},
      {"label":"Разминка","pct":0.40,"reps":8,"weight":"..."},
      {"label":"Разминка","pct":0.55,"reps":5,"weight":"..."},
      {"label":"Разминка","pct":0.70,"reps":3,"weight":"..."},
      {"label":"Разминка","pct":0.80,"reps":"1–2","weight":"..."},
      {"label":"Опц.","pct":0.90,"reps":1,"weight":"..."},
      {"label":"Рабочий","pct":1.00,"reps":"см. план","weight":"..."}
    ]
  }
}
```
**Правила:**
- Источник строк — **Plan** (фильтр по `Week/Day`).
- `tm` — из **TMs** по `Exercise` (колонка «TM (90%)»).
- `recommended = round(step, tm * pct)`; если нет `tm` или `pct` — пусто.
- `units/step` — из **Settings**.

### 5.2 POST `/log`
**Тело (application/json):**
```json
{
  "rows": [
    {
      "week": 1,
      "day": 1,
      "exercise": "Жим лёжа (штанга)",
      "set": 1,
      "repMin": 4,
      "repMax": 6,
      "rirPlan": 2,
      "prescribed": null,
      "reps": 5,
      "weight": 95,
      "rir": 2,
      "note": "-"
    }
  ]
}
```
**Сервер делает:**
- `Timestamp = now()`.
- **Анти‑дубликаты**: окно **±60 секунд** по `(Exercise|Set)` (скан последних ~500 строк).
- `e1RM = weight * (1 + reps/30)`, `TM = 0.9 * e1RM` (если есть `weight` и `reps`).
- Пакетный **append** в **Log** (одним диапазоном).
- **Инкрементальное обновление TMs**: если новый `e1RM` выше, обновить/создать запись.
- **Ответ 200:** `{"ok": true, "added": N, "prs": M}`.

## 6) UX (iPhone 11 first)

- Один экран. Сверху: селекты **Week(1–4)** и **Day(1–4)**.
- Блоки упражнений дня:  
  Заголовок `Exercise — рекоменд.: ХХ (units)` + строка warm‑up (маленьким) + таблица сетов:
  **Set | Min | Max | Plan RIR | Actual Reps | Actual Weight | Actual RIR | Note**.  
  Поле **Actual Weight** — `placeholder = recommended`, если оно есть.
- Кнопка **Save** — фиксирована снизу (sticky).  
- Шрифты ≥16px, тап‑области ≥44pt, без горизонтального скролла.
- Тосты: «Сохранено: N, PR: M»; ошибки — ненавязчиво.

## 8) Интеграция и деплой

### PWA (GitHub Pages)
- В `/pwa/config.json` хранится `{"SCRIPT_URL": "<Web App URL из Apps Script>"}`.

- Вариант 2 (вручную): «Расширения → Apps Script» → вставить файлы → Deploy как **Web app (Anyone with the link)**.
- Вставить URL в `/pwa/config.json`.

## 9) Acceptance Criteria (Definition of Done)

1) **GET /day** возвращает корректный набор сетов дня из **Plan**; `recommended` = `round(step, tm*pct)`; warm‑up присутствует.
2) **POST /log** добавляет строки в **Log**, считает `e1RM/TM`, обновляет **TMs** при PR; повторная отправка идентичных сетов ≤60 сек → `added=0`.
3) **UX:** на iPhone 11 нет горизонтального скролла; `Save` доступна; placeholder рекоменд. веса виден.
4) **Settings:** изменение `UNITS/STEP_*` влияет на `units/step/округление` в ответе `/day` без правки фронта.

## 10) Тренировочный план для листа Plan (4 недели × 4 дня)

### Схема недель (для основных: Жим лёжа, Становая тяга, Присед шир. стойка/бокс)
- **W1:** Top 4–6 @ **0.82 TM**, Back‑off 5–6 @ **0.72**, **RIR=2**
- **W2:** Top 3–5 @ **0.85 TM**, Back‑off 4–6 @ **0.75**, **RIR=2**
- **W3:** Top 2–4 @ **0.88 TM**, Back‑off 3–5 @ **0.80**, **RIR=1**
- **W4 (делоад):** Top 5–6 @ **0.70 TM**, Back‑off 6–8 @ **0.65**, **RIR=2**

**Warm‑up** для основных: гриф (`BAR_WEIGHT`) → 40%×8 → 55%×5 → 70%×3 → 80%×1–2 → (опц.) 90%×1 → рабочий (recommended). Округление — по `step`.

#### День 1 — Жим + плечи + трицепс
- Жим лёжа (штанга): **Top + 3 Back‑off** (по неделе)  
- Жим плеч гантелями (сидя) 3×6–8 @ RIR 1–2  
- Махи в стороны 4×12–20 @ RIR 2  
- Разгибания трицепса (канат) 3×10–15 @ RIR 1–2  
- (Опц.) задняя дельта 2–3×15–20

#### День 2 — Тяга + спина (ноги щадяще)
- Становая тяга (классика): **Top + 3 Back‑off**  
- Тяга в наклоне (опора грудью/машина) 4×6–10  
- Тяга верхнего блока 3×8–12  
- Сгибания ног сидя (машина) 3×10–15  
- Бицепс EZ 2×8–12

#### День 3 — Жим облегчённый + плечи/руки «памп»
- Жим лёжа пауза/узкий 3×6–8 @ ~0.70 TM, RIR 2  
- Арнольд‑пресс 3×10–15  
- Махи в стороны (машина/кроссовер) 3×15–25  
- Обратные разведения 3×15–20  
- Разгибания ног (лёгк./BFR) 3×20–30  
- Трицепс над головой (канат) 2×12–20  
- Кроссовер/кабель на бицепс 2×12–20

#### День 4 — Присед (шир. стойка, бокс) + задняя цепь
- Присед шир. стойка (бокс): **Top + 3 Back‑off**  
- Тяга румынская 3×6–10  
- Жим ногами (ступни высоко) 3×12–20  
- Икроножные 3×8–12  
- Молотковые сгибания 2×10–15

**Межцикловая прогрессия TM:** после W3/W4 — если топ‑сеты выполнялись не тяжелее планового RIR, поднять TM: **+2.5 кг** (верх), **+5 кг** (становая/присед).
