# MESO • Тренировки v3.0

Полностью локальное приложение для отслеживания тренировок с использованием SQL.js и карточной раскладкой для iPhone.

> **Версия 3.0** - Обновлено 11.12.2025  
> Полностью оптимизированное приложение с умной системой TM, микро-коррекцией и улучшенной производительностью

## 🎯 Особенности

### Основные функции
- ✅ **Полностью локальное** - все данные хранятся в браузере (IndexedDB)
- ✅ **Работает офлайн** - не требует интернета после первой загрузки
- ✅ **PWA** - можно установить на iPhone/Android как нативное приложение
- ✅ **SQL.js** - полноценная SQL база данных в браузере
- ✅ **Экспорт/импорт** - резервные копии в JSON формате (с GZIP компрессией)
- ✅ **Веб-шаринг** - экспорт с возможностью прямого шаринга в Files/iCloud
- ✅ **Умный импорт** - слияние данных без перезаписи существующих записей
- ✅ **Автосохранение** - данные сохраняются автоматически при заполнении сетов
- ✅ **Оптимизированная производительность** - батчинг запросов, кеширование DOM, requestAnimationFrame

### Автоматические расчеты
- ✅ **RIR (Reps in Reserve)** - автоматический расчет запаса повторов
- ✅ **RPE (Rate of Perceived Exertion)** - автоматический расчет воспринимаемой нагрузки
- ✅ **e1RM** - расчет максимального веса на 1 повторение (формула Epley: `e1RM = вес × (1 + повторы / 30)`)
- ✅ **Тренировочный максимум (ТМ)** - умная система с EMA-сглаживанием:
  - Автоматический посев TM из качественных сетов (Week 1)
  - EMA-обновление TM (Week 2-3): `TM_new = 0.8 × TM_old + 0.2 × e1RM_session`
  - Лимиты шага: ≤1.5% вверх, ≤1.0-1.5% вниз (ноги до 3%)
  - Анти-тильт защита от резких падений
  - История изменений TM с журналом
  - Ручная блокировка TM (замок 🔒)
- ✅ **Автоматический расчет веса/повторов** - при вводе одного значения автоматически рассчитывается второе на основе ТМ и целевого RIR (только для упражнений типа A)
- ✅ **Микро-автокоррекция** - автоматическая корректировка веса следующего сета на основе отклонения RIR от цели (±2.5%)
- ✅ **Авто-предзаполнение** - первый сет автоматически заполняется при рендере карточки
- ✅ **RIR-first на Week 1** - приоритет целевого RIR над верхней границей повторов
- ✅ **Автоматический расчет TM** - TM рассчитывается автоматически при первом сохранении сета для упражнений типа A

## 🚀 Быстрый старт

1. Откройте `index.html` в браузере (рекомендуется Chrome/Safari)
2. При первом запуске автоматически создастся БД и загрузится план тренировок
3. Выберите неделю (1-4) и день тренировки
4. Введите вес и повторы - остальное считается автоматически
5. TM (тренировочный максимум) рассчитывается автоматически при первом сохранении сета для упражнений типа A
6. Данные сохраняются автоматически при заполнении

## 📱 Использование

### Тренировка

1. Выберите **неделю** (1-4) и **день** тренировки
2. Выберите **дату**
3. **ТМ (тренировочный максимум)** - только для упражнений типа A:
   - **Week 1:** Автоматический посев TM из качественных сетов (93-95% от e1RM)
   - **Week 2-3:** EMA-обновление TM с лимитами шага и анти-тильт защитой
   - **Week 4:** TM не обновляется (делоуд неделя)
   - Можно вручную изменить, нажав на кнопку 🔒 рядом с TM (блокировка)
   - Для аксессуарных упражнений (B, C, D) ТМ не требуется

4. Введите **вес** и **повторы** для каждого сета:
   - **Для типа A:** 
     - При вводе только веса - автоматически рассчитываются повторы (на основе ТМ и целевого RIR)
     - При вводе только повторов - автоматически рассчитывается вес
     - Микро-автокоррекция: если RIR отклоняется от цели, следующий сет автоматически корректируется
   - **Для типов B, C, D:** Просто вводите вес и повторы вручную
   - RIR рассчитывается автоматически и отображается с цветовой индикацией (для всех типов)
   - Первый сет автоматически предзаполняется при открытии дня

5. **Добавление/удаление сетов**:
   - Кнопка "+ Добавить сет" - добавляет новый сет
   - Кнопка ❌ рядом с сетом - удаляет сет (недоступна, если сет только один)

### Подсказки

- **Принять все подсказки** - применяет рекомендации для всех упражнений одним тапом
- **Заполнить** - автоматически подставляет вес и повторы из последней тренировки этого упражнения
- **Рассчитать** - рассчитывает рекомендуемый вес на основе истории и целевого RIR

### Экспорт/импорт

- **Экспорт** - сохраняет всю БД в JSON файл для резервной копии (с GZIP компрессией)
- **Веб-шаринг** - при экспорте предлагается шаринг напрямую в Files/iCloud (если поддерживается)
- **Импорт** - восстанавливает БД из JSON файла
- **Умный импорт** - режим слияния данных без перезаписи существующих записей (показывает diff: добавлено/обновлено/пропущено)
- **Напоминание об экспорте** - приложение напоминает делать резервные копии раз в неделю с опциями:
  - "Отложить на неделю"
  - "Экспорт + Шаринг"

## 📊 Формат данных

### План тренировок (plan.csv)

CSV формат: `day,exercise,setrep,type,rir_w1,rir_w2,rir_w3,rir_w4,note`

Пример:
```csv
1,Жим лежа,4x8,A,3-4,2-3,1-2,0-1,Основное упражнение
```

### Экспорт БД

JSON формат с base64-кодированной БД (GZIP сжатие):
```json
{
  "version": 2,
  "data": "base64-encoded-gzip-database",
  "compressed": true,
  "originalSize": 76541,
  "compressedSize": 7119,
  "timestamp": "2025-12-11T..."
}
```

## 🔧 Технические детали

### Технологии

- **SQL.js** - SQLite в браузере (WebAssembly)
- **IndexedDB** - хранение БД
- **PWA** - Progressive Web App для установки на iPhone/Android
- **Service Worker** - кэширование для офлайн работы
- **CompressionStream API** - сжатие экспортируемых данных
- **Web Share API** - шаринг резервных копий
- **requestAnimationFrame** - оптимизация обновлений DOM

### Хранение данных

- Все данные хранятся в IndexedDB
- БД автоматически сохраняется при каждом изменении
- Данные сохраняются между сессиями
- Автосохранение черновика каждые 5 секунд
- Локальное хранилище для настроек пользователя и черновиков

### Резервные копии

- Экспорт БД в JSON файл (с GZIP компрессией)
- Импорт БД из JSON файла
- Умный импорт с режимом слияния
- Веб-шаринг для прямого сохранения в Files/iCloud
- Рекомендуется делать резервные копии регулярно (приложение напоминает раз в неделю)

### База данных

Схема БД включает:
- `plan` - план тренировок
- `tm` - тренировочные максимумы по упражнениям (с полями `locked`, `source`)
- `tm_log` - журнал изменений TM (для истории)
- `tracker` - история выполненных сетов
- `sessions` - сессии тренировок
- `profile` - профиль пользователя
- `schema_version` - версия схемы БД (для миграций)

Представления (views):
- `v_last_sets` - последние сеты по каждому упражнению
- `v_best_e1rm` - лучшие e1RM по упражнениям
- `v_weekly_volume` - недельный объем нагрузки

### Оптимизация производительности

- **Батчинг INSERT** - множественные INSERT в один запрос
- **Кеширование DOM** - элементы кешируются для уменьшения запросов
- **requestAnimationFrame** - батчинг обновлений карточек
- **Оптимизация проверок** - один запрос вместо N для проверки существующих записей
- **Батчинг TM запросов** - загрузка всех TM одним запросом

### Конфигурация

Все настройки централизованы в `config.js`:
- RIR по типам упражнений для каждой недели
- Шаги веса по типам упражнений
- Лимиты обновления TM
- Параметры EMA-сглаживания
- Параметры посева TM
- Настройки микро-автокоррекции
- Фильтры качества сетов
