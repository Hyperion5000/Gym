# GS Trainer — Google Sheets Edition

Минимальная и удобная система ведения тренировок на базе Google Sheets и Apps Script. Интерфейс запускается прямо в таблице: выбираем неделю и день, вводим факт, всё остальное считается автоматически.

**Версия:** 1.0.0  
**Последнее обновление:** Code review fixes — улучшена обработка ошибок, валидация и безопасность типов данных.

## Структура проекта

- `gas/Code.gs` — основной код Apps Script (меню, API, расчёты, анти-дубли, архивирование, REST API).
- `gas/Sidebar.html` — боковая панель с формой ввода (адаптирована для iPhone 11).
- `gas/SidebarStyles.html` — стили интерфейса (мобильная адаптация, безопасные зоны).
- `gas/SidebarApp.html` — клиентская логика интерфейса (валидация, фильтрация пустых данных).
- `TZ_GS_Trainer_MVP_v2.md` — исходное техзадание.

## Подготовка Google Sheets

### Автоматическая настройка (рекомендуется)

1. Создайте новую таблицу Google Sheets.
2. Откройте `Extensions → Apps Script` и установите код (см. раздел "Установка Apps Script").
3. Вернитесь в таблицу и обновите страницу.
4. В меню выберите **GS Trainer → Подготовить таблицу** — автоматически создадутся все необходимые листы с правильной структурой.

### Ручная настройка (опционально)

Если нужно настроить вручную, создайте листы со следующими колонками:

- **Settings** (B:C) — настройки:
  - `UNITS` — единицы измерения (`kg` или `lb`)
  - `STEP_KG` — шаг округления для кг (по умолчанию `2.5`)
  - `STEP_LB` — шаг округления для фунтов (по умолчанию `5`)
  - `BAR_WEIGHT` — вес пустого грифа (по умолчанию `20`)
  - `ARCHIVE_DAYS` — через сколько дней архивировать (по умолчанию `60`)

- **Plan** — тренировочный план:
  - `Key | Week | Day | Exercise | Set # | Rep Min | Rep Max | RIR | Prescribed | PctTM`

- **Log** — журнал тренировок:
  - `Timestamp | Week# | Day# | Exercise | Set # | Rep Min | Rep Max | RIR план | Prescribed | Actual Reps | Actual Weight | Actual RIR | e1RM | TM (90%) | Updated | Notes`

- **TMs** — тренировочные максимумы:
  - `Exercise | last e1RM | TM (90%) | Updated`

- **Archive** — архив старых записей (копия структуры `Log`)

## Установка Apps Script

1. В таблице: `Extensions → Apps Script`.
2. Удалите созданный по умолчанию `Code.gs`.
3. Создайте файлы и вставьте содержимое из каталога `gas/`:
   - `Code.gs`
   - `Sidebar.html`
   - `SidebarStyles.html`
   - `SidebarApp.html`
4. Сохраните проект (например, `GS Trainer MVP`).
5. Вернитесь в таблицу (Sheet) и обновите страницу.

## Использование

### Первоначальная настройка

1. После установки кода в меню появится пункт **GS Trainer**.
2. Запустите **GS Trainer → Подготовить таблицу** — создадутся листы и настройки по умолчанию (⚠️ команда очищает существующие данные).
3. Загрузите тренировочный цикл командой **GS Trainer → Импортировать план 4×4** (перезаписывает лист `Plan`).
   - План включает 4 недели × 4 дня с прогрессией нагрузки
   - Упражнения: жим лёжа, становая, присед, брусья, аксессуары для плеч/рук/спины
   - Реабилитационные упражнения для коленей и предплечий
4. Выполните **GS Trainer → Проверить конфигурацию** — убедитесь, что структура корректна (подсветит ошибки красным).

### Работа с тренировками

1. Нажмите **GS Trainer → Открыть тренировку** (или **GS Trainer → Open Sidebar**).
2. В боковой панели:
   - Выберите **Неделю** (1-4) и **День** (1-4)
   - Нажмите **Загрузить** — отобразятся упражнения на выбранный день
3. Для каждого упражнения:
   - Показана разминка (если есть TM для упражнения)
   - Рекомендованный вес (если есть TM и процент в плане)
   - План: сет, диапазон повторов, RIR
4. Заполните фактические значения:
   - **Вес** (кг/фунты) — автоматически округляется до шага
   - **Повторы** (1-30)
   - **RIR** (0-3)
   - **Заметка** (опционально, до 160 символов)
5. Нажмите **Сохранить** — скрипт:
   - ✅ Проверит дубли `(Exercise|Set)` в окне ±60 секунд
   - ✅ Валидирует данные (week/day 1-4, reps 1-30, RIR 0-3, weight ≥ 0)
   - ✅ Сохранит только строки с данными (пустые пропускаются)
   - ✅ Посчитает `e1RM = weight × (1 + reps/30)`
   - ✅ Посчитает `TM = e1RM × 0.9`
   - ✅ Обновит лист `TMs` (только если новый e1RM больше текущего)
   - ✅ Зафиксирует PR (Personal Records)
   - ✅ Покажет сообщение «Сохранено: N, PR: M»

## Функции меню

- **Подготовить таблицу** — автоматически создаёт все листы с правильной структурой и настройками по умолчанию.
- **Импортировать план 4×4** — заполняет лист `Plan` тренировочным циклом (4 недели × 4 дня) с прогрессией нагрузки.
- **Проверить конфигурацию** — валидирует структуру таблицы, подсвечивает ошибки красным цветом.
- **Open Sidebar** / **Открыть тренировку** — открывает боковую панель для ввода данных.
- **Archive Old Logs** — переносит записи старше `ARCHIVE_DAYS` дней из `Log` в `Archive`.

## REST API (опционально)

Для интеграции с внешними приложениями доступны REST эндпоинты:

### Настройка Web App

1. В Apps Script: `Deploy → New deployment`
2. Тип: `Web app`
3. Execute as: `Me`
4. Who has access: `Anyone` (или `Anyone with Google account`)
5. Скопируйте URL развёртывания

### Эндпоинты

**GET `/exec?fn=day&week={1..4}&day={1..4}`**
- Возвращает план на день, настройки и текущие TMs
- Формат: `{ ok: true, plan: [...], settings: {...}, tms: {...} }`

**GET `/exec?fn=settings`**
- Возвращает настройки системы
- Формат: `{ ok: true, units: "kg", step: 2.5, barWeight: 20, archiveDays: 60 }`

## Автоматизация (опционально)

- **Time-driven trigger** для `archiveOldLogs` — настройте в Apps Script (`Triggers → Add trigger`) для автоматической архивации (например, раз в неделю).
- **Looker Studio (Data Studio)** — подключите Google Sheets как источник данных для визуализации прогресса.
- **Telegram-бот** — используйте REST API для отправки данных из бота.
- **Google Calendar** — можно добавить напоминания о тренировках через Apps Script.

Логика централизована в Apps Script, что упрощает интеграцию с внешними сервисами.

## Советы

- **Поддержка плана**: Поддерживайте шаблон `Plan` на 4 недели × 4 дня. При необходимости создавайте новые циклы, копируя строки и обновляя недели/дни.
- **Настройки**: Изменение `UNITS` и `STEP_*` в `Settings` автоматически влияет на расчёт рекомендованного веса и округления.
- **TM (Training Max)**: Автоматически рассчитывается как 90% от лучшего e1RM. Обновляется только при улучшении результата.
- **Разминка**: Генерируется автоматически для упражнений с TM (пустой гриф → 40% → 55% → 70% → 80% → 90% → рабочий вес).
- **Архивация**: Регулярно архивируйте старые записи для поддержания производительности (настройте trigger или запускайте вручную).

## Технические детали

- **Язык**: Google Apps Script (JavaScript ES5+)
- **Хранение**: Google Sheets (5 листов: Settings, Plan, Log, TMs, Archive)
- **Интерфейс**: HtmlService (Sidebar в Google Sheets)
- **API**: REST API через Web App deployment
- **Формулы**:
  - `e1RM = weight × (1 + reps/30)` (Epley formula)
  - `TM = e1RM × 0.9` (90% от e1RM)
  - Округление до шага: `round(value / step) × step`
